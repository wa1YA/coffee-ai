# 咖啡大师 AI - 从零到上线完整教程

> 手把手教你把这个项目部署到公网，获得一个可访问的 URL。
> 四条路线，从易到难，选一条适合你的。

## 上线前准备（5分钟）

### 1. 获取 DeepSeek API Key

打开 https://platform.deepseek.com → 注册/登录 → API Keys → 创建新 Key → 复制

费用极低：deepseek-chat 模型 1元/百万token，充10块钱能用很久。

### 2. 在你的电脑上用 Git 管理项目

```bash
# 进入项目目录
cd "D:\桌面\611面试\coffee-ai-deploy"

# 初始化 Git
git init
git add .
git commit -m "init: 咖啡大师AI问答助手"
```

### 3. 创建 .env 文件（本地测试用）

复制 `.env.example` 为 `.env`，填入你的 API Key：
```
DEEPSEEK_API_KEY=sk-你的真实key
```

然后本地测试一下：
```bash
pip install -r requirements.txt
python app.py
# 浏览器打开 http://localhost:5000，确认能正常问答
```

## 路线一：Railway（最简单，推荐首选）

> Railway 是海外 PaaS 平台，免费额度 5 美元/月（够跑一个月），支持 GitHub 自动部署。
> 它在你 push 代码后自动构建并上线，完全不需要管服务器。

### 步骤

**1. 把代码推到 GitHub**

```bash
# 在 GitHub 上新建仓库，名字随意（如 coffee-ai）
# 然后：
git remote add origin https://github.com/你的用户名/coffee-ai.git
git branch -M main
git push -u origin main
```

**2. 注册 Railway**

打开 https://railway.app → Sign Up → 用 GitHub 账号登录

**3. 创建项目并部署**

- 点击 `New Project` → `Deploy from GitHub Repo`
- 选择你刚才推的 `coffee-ai` 仓库
- Railway 会自动检测到 `Procfile`，知道这是一个 Python Web 应用
- 它会自动 `pip install -r requirements.txt` 然后用 `gunicorn` 启动

**4. 设置环境变量**

在 Railway 项目面板 → `Variables` → 添加：

| 变量名 | 值 |
|--------|-----|
| `DEEPSEEK_API_KEY` | sk-你的key |
| `SECRET_KEY` | 随便打一串乱码 |
| `HF_ENDPOINT` | https://hf-mirror.com |

**5. 等待部署完成**

Railway 会自动构建 Docker 镜像并启动。第一次部署因为要下载模型，大概需要 5-10 分钟。

部署完成后，Railway 会给你一个 URL：`https://coffee-ai.up.railway.app`

打开这个 URL → 你的项目上线了！

**6. 后续更新**

每次修改代码，只需要：
```bash
git add . && git commit -m "更新xxx" && git push
```
Railway 会自动重新部署。

**优点**：零服务器运维，免费额度够用，自动 HTTPS
**缺点**：国内访问可能慢（海外服务器），免费额度用完后需付费

## 路线二：阿里云 + 宝塔面板（国内最实用）

> 适合有国内云服务器的同学。如果你还没有服务器，阿里云学生优惠一年不到 100 块。

### 步骤

**1. 购买阿里云 ECS**

- 打开 https://ecs.console.aliyun.com
- 学生认证后有优惠（约 99元/年）
- 配置选最低配即可：2核4G，CentOS 7.9 或 Ubuntu 22.04
- 地区选离你最近的（深圳/广州）
- 系统选 Ubuntu 22.04（后面步骤基于此）

**2. 安装宝塔面板**

用 SSH 连接服务器后：
```bash
# 安装宝塔面板
wget -O install.sh http://download.bt.cn/install/install-ubuntu_6.0.sh
sudo bash install.sh

# 安装完成后会显示：
# 面板地址: http://你的IP:8888/xxxxxxxx
# 用户名: xxxxxx
# 密码: xxxxxx
```

**3. 宝塔面板安装环境**

浏览器打开宝塔面板地址 → 登录 → 会提示安装 LNMP/LAMP

只需安装：
- Nginx（Web 服务器）
- Python 管理器（软件商店搜索安装）

**4. 上传项目代码**

在宝塔面板 → 文件 → 进入 `/www/wwwroot/` → 新建目录 `coffee-ai`

有两种方式上传代码：
- 方式A：宝塔面板文件管理器直接拖拽上传（适合不熟悉 Git 的）
- 方式B：SSH 连上服务器 `git clone https://github.com/你的用户名/coffee-ai.git`

**5. 用 Python 项目管理器部署**

宝塔面板 → 软件商店 → Python 项目管理器 → 添加项目：

| 字段 | 值 |
|------|-----|
| 项目路径 | `/www/wwwroot/coffee-ai` |
| 启动文件 | `wsgi.py` |
| 启动方式 | `gunicorn` |
| 端口 | `5000` |
| Python 版本 | `3.12` |
| 框架 | `Flask` |
| 运行用户 | `www` |

然后点击"模块"标签 → 安装依赖 → 选择 `requirements.txt`

环境变量设置：
```
DEEPSEEK_API_KEY=sk-你的key
SECRET_KEY=随便打一串乱码
HF_ENDPOINT=https://hf-mirror.com
```

**6. 配置 Nginx 反向代理**

宝塔面板 → 网站 → 添加网站 → 填写：

| 字段 | 值 |
|------|-----|
| 域名 | 你的域名（没有就填服务器IP） |
| PHP版本 | 纯静态 |

然后点击网站设置 → 反向代理 → 添加反向代理：

| 字段 | 值 |
|------|-----|
| 代理名称 | coffee-ai |
| 目标URL | `http://127.0.0.1:5000` |
| 发送域名 | `$host` |

保存后，你的项目就上线了！打开 `http://你的域名或IP` 就能访问。

**7. 配置 SSL 证书（可选但建议）**

宝塔面板 → 网站 → 设置 → SSL → Let's Encrypt → 一键申请

这样就有 HTTPS 了。

**优点**：国内访问快，完全可控，还能部署其他项目
**缺点**：需要买服务器、需要配置

## 路线三：Docker 自部署（你有服务器的话最快）

> 如果你已经有任何 Linux 服务器、已安装 Docker，这条路线最快。

### 步骤

**1. 在服务器上安装 Docker**

```bash
curl -fsSL https://get.docker.com | bash
```

**2. 上传项目到服务器**

```bash
# 在你电脑上：把项目打包
cd "D:\桌面\611面试\coffee-ai-deploy"
scp -r . root@你的服务器IP:/opt/coffee-ai/

# 或者用 git clone
ssh root@你的服务器IP
cd /opt
git clone https://github.com/你的用户名/coffee-ai.git
```

**3. 配置环境变量**

```bash
cd /opt/coffee-ai

# 创建 .env
cat > .env << 'EOF'
DEEPSEEK_API_KEY=sk-你的key
SECRET_KEY=随机乱码
HF_ENDPOINT=https://hf-mirror.com
EOF
```

**4. 一键启动**

```bash
docker compose up -d
```

项目就跑起来了：`http://你的服务器IP:5000`

**5. 配置 Nginx 反向代理 + SSL**

```bash
# 安装 Nginx
apt install nginx -y

# 配置
cat > /etc/nginx/sites-available/coffee-ai << 'EOF'
server {
    listen 80;
    server_name 你的域名或IP;

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
EOF

ln -s /etc/nginx/sites-available/coffee-ai /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx
```

**优点**：最快部署，一行命令搞定，方便迁移
**缺点**：需要会 Docker 基础

## 路线四：PythonAnywhere（免费，无需信用卡）

> 纯免费方案，适合只是想有个链接可以展示。

### 步骤

**1. 注册 PythonAnywhere**

打开 https://www.pythonanywhere.com → 注册免费账号（Beginner 账号）

**2. 上传代码**

PythonAnywhere → Files 标签 → 进入 `/home/你的用户名/` → 创建目录 `coffee-ai`

点击 Upload a file → 把项目文件一个个上传上去（免费账号不支持 Git）

**3. 安装依赖**

PythonAnywhere → Consoles → 打开一个 Bash console：
```bash
cd ~/coffee-ai
pip install --user flask sentence-transformers faiss-cpu numpy scikit-learn openai python-dotenv
```

**4. 配置 Web App**

PythonAnywhere → Web 标签 → Add a new web app → 选择 Flask + Python 3.12

修改 WSGI 配置文件，指向我们的 app：
```python
import sys
sys.path.insert(0, '/home/你的用户名/coffee-ai')

from app import create_app
application = create_app()
```

**5. 设置环境变量**

在 WSGI 配置文件顶部：
```python
import os
os.environ["DEEPSEEK_API_KEY"] = "sk-你的key"
os.environ["SECRET_KEY"] = "随机乱码"
os.environ["HF_ENDPOINT"] = "https://hf-mirror.com"
```

**6. 重新加载**

点击 Web 标签的 Reload 按钮。

你的项目上线了：`https://你的用户名.pythonanywhere.com`

**优点**：完全免费，无需信用卡
**缺点**：免费版有访问限制、不能后台任务、性能一般、域名是 pythonanywhere.com

## 常见问题

**Q: 模型下载太慢或失败？**
A: 已经配置了 `HF_ENDPOINT=https://hf-mirror.com`，这是国内镜像。如果还慢，可以先在本地下载好模型文件，直接复制到服务器的模型缓存目录。

**Q: 内存不够？**
A: sentence-transformers 模型加载需要约 500MB 内存，加上应用本身，建议服务器至少 2GB 内存。如果不够，可以把 EmbeddingEngine 的 transformer 模式改成 tfidf 模式（已在代码中实现自动降级）。

**Q: API Key 安全吗？**
A: 使用了 `.env` 文件管理密钥，`.gitignore` 已排除 `.env`。生产环境通过系统环境变量注入，不会出现在代码里。Railway/Docker Compose 都有专门的变量管理。
