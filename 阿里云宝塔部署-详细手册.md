# 阿里云 + 宝塔面板 上线操作手册

> 目标：把咖啡大师 AI 部署到阿里云，获得一个公网可访问的地址。
> 预计总耗时：1-2小时（大部分时间在等安装）。

---

## 前言：这条路涉及什么

```
你电脑上的代码
     │
     ▼  git push
  GitHub（代码仓库）
     │
     ▼  git clone
阿里云 ECS 服务器（Ubuntu 22.04）
     │
     ▼  安装宝塔面板
宝塔面板（Web 可视化管理）
     │
     ▼  一键部署 Python 项目
公网 URL：http://你的IP
```

---

## 第一步：购买阿里云服务器（20分钟）

### 1.1 注册 + 学生认证

1. 打开 https://aliyun.com
2. 注册账号（已有淘宝账号可直接登录）
3. 右上角头像 → 实名认证 → 完成个人实名
4. 搜索"学生优惠"或打开 https://university.aliyun.com
5. 完成学生认证（需要学信网信息，广东海洋大学在库里）

### 1.2 购买 ECS

学生认证通过后，能看到学生专享价（约 99元/年）。

选择配置：

| 配置项 | 选择 | 原因 |
|--------|------|------|
| 地域 | **深圳** 或 **广州** | 你在广东离得近 |
| 实例规格 | **2核4G** 最低配 | 我们的项目够用（模型占~1G内存） |
| 镜像 | **Ubuntu 22.04 LTS** | 后面所有命令基于此 |
| 系统盘 | 40GB 高效云盘 | 默认即可 |
| 带宽 | 按量计费 或 1Mbps | 个人项目够用 |
| 购买时长 | **1年** | 学生优惠通常按年买 |

> 如果学生优惠过期或不能用，去 ecs.console.aliyun.com → 创建实例 → 选"突发性能实例 t6"（2核2G就行），按量付费一个月也就三四十块。

### 1.3 获取服务器信息

购买成功后，进入 **ECS 控制台** → 实例列表，你会看到：

- **公网 IP**：例如 `47.107.xxx.xxx` —— 记下来，这就是你的网址
- **实例状态**：运行中

点击实例 ID 进入详情 → 左侧"重置实例密码" → 设置 root 密码（**记下来**）。

然后点击"远程连接" → 确认你能进入终端（黑窗口）。

## 第二步：连接服务器（5分钟）

### 2.1 在你的 Windows 电脑上

打开 PowerShell 或 CMD：

```bash
ssh root@47.107.xxx.xxx
```

第一次会提示 `Are you sure you want to continue connecting?` → 输入 `yes`

输入你刚才设置的密码。看到 `root@xxx:~#` 就进去了。

### 2.2 先做安全基础配置

```bash
# 更新系统
apt update && apt upgrade -y
```

---

## 第三步：安装宝塔面板（10分钟）

### 3.1 安装命令

在 SSH 终端里执行：

```bash
wget -O install.sh http://download.bt.cn/install/install-ubuntu_6.0.sh && sudo bash install.sh
```

安装过程中会提示 `Do you want to install Bt-Panel to the /www directory now?` → 输入 `y` 回车。

### 3.2 等待安装完成

大约 5-10 分钟。最后会显示：

```
==================================================================
外网面板地址: http://47.107.xxx.xxx:8888/abc12345
内网面板地址: http://192.168.0.1:8888/abc12345
username: xxxxxxxxxx
password: xxxxxxxxxx
==================================================================
```

**把这三行信息复制保存下来！** 这是你访问宝塔的凭证。

### 3.3 开放防火墙端口

阿里云默认有安全组限制，需要手动放行端口。

回到阿里云控制台 → ECS → 实例 → 点击你的实例 → **安全组** 标签 → 点击安全组ID → **入方向** → **手动添加**：

| 端口 | 授权对象 | 描述 |
|------|---------|------|
| 8888 | 0.0.0.0/0 | 宝塔面板 |
| 80 | 0.0.0.0/0 | 网站 HTTP |
| 443 | 0.0.0.0/0 | 网站 HTTPS |
| 5000 | 0.0.0.0/0 | Flask 开发端口（临时用） |

> 注意：这个操作在阿里云网页控制台，不在 SSH 终端里。

### 3.4 登录宝塔面板

浏览器打开宝塔的 `外网面板地址`，输入用户名和密码，进入宝塔管理界面。

首次登录会弹出一个推荐安装套件的窗口，**不要全装**！

只看这个窗口 → 选择 **LNMP** 旁边：

- Nginx：勾选（安装最新的稳定版）
- MySQL：**不勾选**（我们的项目不需要数据库）
- PHP：**不勾选**（我们用 Python）
- phpMyAdmin：**不勾选**
- Pure-Ftpd：**不勾选**

点击 "一键安装" → 等 Nginx 装完（约2分钟）。

## 第四步：上传项目代码（10分钟）

### 4.1 先把代码推到 GitHub

在你电脑上：

```bash
cd "D:\桌面\611面试\coffee-ai-deploy"

# 确认 .env 不在 git 追踪里

# 在 GitHub 网页上创建一个新仓库（如 coffee-ai）
# 然后：
git remote add origin https://github.com/你的用户名/coffee-ai.git
git branch -M main
git push -u origin main
```

### 4.2 服务器上拉取代码

SSH 连回服务器：

```bash
# 安装 git
apt install git -y

# 进入网站目录
cd /www/wwwroot

# 克隆代码
git clone https://github.com/你的用户名/coffee-ai.git

# 进入项目
cd coffee-ai
ls
# 应该看到 app.py, rag_engine.py, agent.py, data/, templates/...
```

### 4.3 创建 .env 环境变量文件

```bash
cd /www/wwwroot/coffee-ai

cat > .env << 'EOF'
DEEPSEEK_API_KEY=sk-你的真实key
DEEPSEEK_BASE_URL=https://api.deepseek.com
LLM_MODEL=deepseek-chat
SECRET_KEY=随便打一串乱码比如jad8fh3k2l1mn9pqw
HF_ENDPOINT=https://hf-mirror.com
HOST=0.0.0.0
PORT=5000
EOF
```

验证一下：
```bash
cat .env
# 确认内容正确
```

## 第五步：配置 Python 环境（15分钟）

### 5.1 安装 Python 3.12

```bash
# 添加 deadsnakes PPA（提供更新的 Python 版本）
add-apt-repository ppa:deadsnakes/ppa -y
apt update
apt install python3.12 python3.12-venv python3.12-dev python3-pip -y
```

### 5.2 创建虚拟环境

```bash
cd /www/wwwroot/coffee-ai

# 创建虚拟环境
python3.12 -m venv venv

# 激活
source venv/bin/activate

# 现在提示符前面应该有 (venv)

# 安装依赖
pip install --upgrade pip
pip install -r requirements.txt
```

这一步比较慢（要下载 sentence-transformers 的模型），约 5-10 分钟。

### 5.3 测试一下

```bash
# 确保在 venv 环境中
cd /www/wwwroot/coffee-ai
python app.py
```

正常的话你会看到：
```

==================================================
[LLM] [OK] deepseek-chat @ https://api.deepseek.com
```

按 `Ctrl+C` 停掉。确认能正常运行即可进入下一步。

## 第六步：用 gunicorn 生产运行（10分钟）

直接用 `python app.py` 是开发模式，生产环境要用 gunicorn。

### 6.1 手动测试 gunicorn

```bash
cd /www/wwwroot/coffee-ai
source venv/bin/activate

gunicorn wsgi:app --bind 0.0.0.0:5000 --workers 2
```

确认能启动后 `Ctrl+C` 停掉。

### 6.2 创建 systemd 服务（保持后台运行）

```bash
cat > /etc/systemd/system/coffee-ai.service << 'EOF'
[Unit]
Description=咖啡大师 AI 问答助手
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/www/wwwroot/coffee-ai
EnvironmentFile=/www/wwwroot/coffee-ai/.env
ExecStart=/www/wwwroot/coffee-ai/venv/bin/gunicorn wsgi:app --bind 0.0.0.0:5000 --workers 2 --timeout 120 --access-logfile /var/log/coffee-ai-access.log --error-logfile /var/log/coffee-ai-error.log
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF
```

启动服务：

```bash
systemctl daemon-reload
systemctl enable coffee-ai
systemctl start coffee-ai
```

检查状态：

```bash
systemctl status coffee-ai
# 看到 active (running) 就对了

# 测试一下
curl http://localhost:5000/health
# 返回 {"status":"ok","llm":true}
```

以后更新代码后重启：

```bash
systemctl restart coffee-ai
```

## 第七步：配置 Nginx 反向代理（10分钟）

现在服务跑在 `http://localhost:5000`，但对外暴露的是 80 端口。

### 7.1 在宝塔面板配置

1. 宝塔面板 → 网站 → **添加站点**
2. 域名栏填：`47.107.xxx.xxx`（你的服务器公网IP）
   - 如果你有域名，就填域名（需要先去 DNS 服务商把域名 A 记录指向这个 IP）
3. PHP 版本选：**纯静态**
4. 点击提交

### 7.2 配置反向代理

宝塔面板 → 网站 → 找到刚建的站点 → 点击 **设置** → **反向代理** 标签 → **添加反向代理**：

| 字段 | 值 |
|------|-----|
| 代理名称 | coffee-ai |
| 目标URL | `http://127.0.0.1:5000` |
| 发送域名 | `$host` |

其他保持默认，点击提交。

### 7.3 解析域名（如果你有域名）

去你的域名服务商（阿里云/腾讯云/其他）：

- 添加一条 **A 记录**
- 主机记录：`@` 或 `www`
- 记录值：你的服务器公网 IP
- 保存，等 1-10 分钟生效

## 可能遇到的报错与解决

| 报错 | 原因 | 解决 |
|------|------|------|
| `ssh: connect to host port 22` | 安全组没开22端口 | 阿里云控制台 → 安全组 → 放行 22 |
| `wget: command not found` | Ubuntu 最小化没装 | `apt install wget -y` |
| `ModuleNotFoundError: No module named 'xxx'` | 依赖没装全 | `pip install -r requirements.txt` |
| gunicorn 启动失败 | 环境变量不对 | 检查 `.env` 文件中的 API Key 是否完整 |
| 浏览器访问不了 | 安全组没开放80端口 | 阿里云控制台放行 80 端口 |
| 模型下载慢 | HuggingFace 连不上 | 已设了 `HF_ENDPOINT=https://hf-mirror.com`，确保 `.env` 里有这一行 |
| 服务器内存跑满 | 2G内存可能吃紧 | `systemctl restart coffee-ai` 释放；或加 swap：`fallocate -l 2G /swapfile && chmod 600 /swapfile && mkswap /swapfile && swapon /swapfile` |
